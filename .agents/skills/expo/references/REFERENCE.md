## Workflow shorthand

```
create-expo-app  →  prebuild (optional)  →  eas build  →  eas submit
                                       ↘                ↑
                                        expo-dev-client (daily dev)
                                       ↘
                                        eas update (JS-only patches)
```

## Key surfaces

- **Config**: `app.config.ts` (or `app.json`), `eas.json`
- **Routing**: `app/_layout.tsx`, `app/[id].tsx`, `app/(group)/`
- **Native customisation**: config plugins in `plugins/`, local Expo modules in `modules/`
- **Build artefacts**: `ios/` and `android/` (generated by `npx expo prebuild`)
- **Env**: `.env`, `EXPO_PUBLIC_*`, EAS Secrets, EAS env vars

## When NOT to open this directory

- Pure React UI tweaks unrelated to mobile → `react`
- Backend / server work → relevant backend skill
- TypeScript type-system puzzles → `typescript`
