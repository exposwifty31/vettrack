63 — Offline Cold-Start Auth — Hard-Fail Session Restore (Production-Ready)

⸻

What & Why

When the app starts without network (cold-start offline), Clerk’s useUser() never resolves isLoaded: true because it cannot reach Clerk servers. ProtectedRoute and RootRoute block on !isLoaded and render an infinite spinner.

Root cause:
ClerkAuthProviderInner gates initialization behind if (!isLoaded) return, and downstream guards rely solely on Clerk state with no awareness of offline restoration.

Fix:
Persist a session snapshot after successful auth. On cold-start offline, restore synchronously before first render under strict hard-fail rules. All render guards must respect isOfflineSession.

⸻

PRIMARY OBJECTIVE — DEPLOYMENT READINESS

This implementation MUST be production-ready:
	•	Deterministic behavior (no race conditions)
	•	No infinite loading states under any condition
	•	Explicit failure states (no silent failures)
	•	Stable under unstable networks and reconnection
	•	Security preserved with controlled degradation

If a tradeoff exists:
→ Prefer predictability and reliability over convenience

⸻

Strategy Selection

Option 1: Synchronous Init + Hard-Fail Restore (SELECTED)
Option 2: Service Worker JWT Intercept (REJECTED — fragile, unsafe)
Option 3: Offline Timeout UI (REJECTED — insufficient)

Key decisions:
	1.	Synchronous initialization (useState lazy init)
	2.	Explicit guard updates across entire app
	3.	Hard-fail validation (no fallback paths)
	4.	Clerk remains source of truth when online

⸻

Hard-Fail Rules (Non-Negotiable)

restoreOfflineSession() returns valid data ONLY if ALL:
	1.	localStorage key "vt_session" exists
	2.	token exists and non-empty
	3.	Date.now() < tokenExp (NOT expired)
	4.	Date.now() - lastActiveAt < 24h
	5.	status === "active"

Else:
→ return null
→ show sign-in

NO EXCEPTIONS

⸻

Critical Architectural Constraint

Offline restore MUST be synchronous (before first render).

DO NOT use useEffect for initial auth.

3. Synchronous Restore (CRITICAL)

Inside ClerkAuthProviderInner:

const initialOfflineSession =
  !navigator.onLine ? restoreOfflineSession() : null;

const [state, setState] = useState(() => {
  if (initialOfflineSession) {
    setAuthState({
      userId: initialOfflineSession.userId,
      email: initialOfflineSession.email,
      name: initialOfflineSession.name,
      bearerToken: initialOfflineSession.token
    });

    return {
      isLoaded: true,
      isSignedIn: true,
      isOfflineSession: true,
      user: {
        id: initialOfflineSession.userId,
        email: initialOfflineSession.email,
        name: initialOfflineSession.name,
        role: initialOfflineSession.role,
        status: initialOfflineSession.status
      }
    };
  }

  return {
    isLoaded: false,
    isSignedIn: false,
    isOfflineSession: false
  };
});
