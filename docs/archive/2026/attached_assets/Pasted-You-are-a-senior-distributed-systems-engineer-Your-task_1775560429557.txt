Your task is to implement and validate a strict consistency scenario involving:
- race conditions
- offline updates
- conflict resolution

Initial state:
equipment.status = "AVAILABLE"
equipment.version = 1

Technician A (offline)
Technician B (online)

2. B performs:
   AVAILABLE → IN_USE
   Server updates:
     status = IN_USE
     version = 2

3. A (still offline) performs:
   AVAILABLE → IN_USE
   This is stored locally (simulate Dexie queue)

4. A reconnects and sends the update

CRITICAL REQUIREMENT:

The server MUST detect version mismatch and reject A's update.

VALID OUTCOME (ONLY THIS IS CORRECT):

- Server rejects A's request (version mismatch)
- Client performs rollback
- Client reflects true server state (IN_USE from B)
- Client logs or returns a clear conflict explanation

INVALID OUTCOMES (MUST FAIL TESTS):

- Server accepts A's update (overwrites B)
- System ends in inconsistent or duplicated state

updateEquipment(currentState, incomingUpdate)
