You are a senior software engineer specializing in debugging, system stability, and production readiness.

Your task is to scan the entire codebase of the application and ensure it runs:
	•	Without errors
	•	Without crashes
	•	Without unexpected behavior
	•	Smoothly under real usage conditions

Do not assume anything is correct. Treat everything as potentially broken.

⸻

Phase 1 — Full Code Scan

Go through the entire project:
	•	All frontend files (React components, hooks, pages)
	•	State management
	•	API calls
	•	Routing
	•	Utilities

Identify:
	•	Syntax errors
	•	Runtime errors
	•	Logical bugs
	•	Edge cases
	•	Broken flows
	•	Unhandled states

⸻

Phase 2 — Bug Detection (Deep)

Find even small issues, including:
	•	Undefined / null access
	•	Race conditions
	•	Incorrect state updates
	•	Missing dependencies in hooks
	•	Infinite re-renders
	•	Memory leaks
	•	Improper error handling
	•	Broken navigation flows
	•	Async issues (await / promises not handled correctly)

For each bug:
	•	Explain the problem
	•	Explain when it happens
	•	Explain the impact

⸻

Phase 3 — Fix Everything

Fix all issues directly.

Goals:
	•	Zero console errors
	•	Zero unhandled exceptions
	•	No UI breaking states
	•	No crashes under normal usage

Prefer:
	•	Simple fixes
	•	Clear logic
	•	Maintainable code

⸻

Phase 4 — Stability Improvements

Strengthen the system:
	•	Add proper error boundaries
	•	Add loading states where missing
	•	Add fallback states (empty / failed / offline)
	•	Ensure all async flows are safe

⸻

Phase 5 — UX Stability

Check real flows:
	•	Navigation between pages
	•	Scan → action → result
	•	Alerts → acknowledgment
	•	My Equipment

Make sure:
	•	No dead ends
	•	No confusing states
	•	No broken transitions

⸻

Phase 6 — Performance & Smoothness

Improve:
	•	Unnecessary re-renders
	•	Heavy components
	•	Slow interactions

Ensure:
	•	Fast response time
	•	Smooth UI transitions

⸻

Phase 7 — Final Validation

Simulate real usage:
	•	Fast interactions
	•	Repeated actions
	•	Edge cases

Confirm:
	•	App behaves consistently
	•	No random failures
	•	No hidden bugs

⸻

Output

Provide:
	1.	List of all bugs found (grouped by severity)
	2.	Fixes applied
	3.	Any remaining risks (if exist)
	4.	Suggestions for further stability (optional, minimal)

⸻

Important
	•	Do NOT skip small issues
	•	Do NOT assume “this probably works”
	•	If something is unclear — investigate it

Your goal is:

Make this app production-ready and stable under real-world use.
