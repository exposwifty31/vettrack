## 1. Overview

VetTrack's design system is an **iOS-native, clinical-calm** token layer laid over a shadcn/Tailwind component base. It renders a veterinary-ICU equipment-tracking app across a web shell and a Capacitor native shell (iPhone + iPad), in light and dark, in **Hebrew-first RTL** and English LTR.

Governing principles:

- **Ivory base / iOS grouped canvas.** The light background is Apple's grouped-table `#F2F2F7` (`--background: 240 24% 96%`); surfaces are pure white on light, `#1C1C1E` on dark. Chrome and hero surfaces use a brand navy `#0b1021`.
- **Clinical-calm aesthetic.** A 60/30/10 split: indigo brand (`--brand #4f46e5`) for nav/headers/primary CTAs; neutral iOS grays for structure; a reserved teal-forest **action** accent (`--action #2f6f5e`) used *only* for scan/confirm affordances, never as brand.
- **Five-level criticality.** A first-class status enum — `ok / issue / maintenance / sterilized / stale / unknown` — each with a hue token plus a `-bg / -fg / -border` pill-fill triad tuned to clear WCAG AA on its own tint, in both themes. (The "five-level ICU criticality" color assignments are a frozen contract; see §8.)
- **Hebrew-first RTL.** Layout uses logical properties (`border-inline-start`, `ms-auto`, `text-start`) and the `tailwindcss-rtl` plugin so every primitive mirrors without per-file rewrites.
- **WCAG 2.1 AA.** Muted foregrounds are deliberately darkened past raw iOS values to clear 4.5:1 on the ivory canvas; a zero-specificity `:focus-visible` fallback guarantees a visible ring on every interactive element; status is never encoded by color alone in the primitives.

Tokens are consumed three ways: **HSL triplets** feed `hsl(var(--name))` (semantic shadcn colors, status hues); **RGB channels** feed `rgb(var(--name) / <alpha-value>)` (ivory / emergency / offline / sys families, for Tailwind opacity support); and **whole values** (`#hex`, `rgba()`, `px`, `ms`, `cubic-bezier`) are referenced directly via `var(--name)`.

## 3. Tailwind extensions (`tailwind.config.ts`)

- **`darkMode: ["class"]`**; content globs `./index.html`, `./src/**/*.{ts,tsx}`; plugins `tailwindcss-animate` + `tailwindcss-rtl`.
- **`screens`: none extended** — Tailwind default breakpoints (`sm 640 / md 768 / lg 1024 / xl 1280 / 2xl 1536`) are used as-is.

**Custom colors** (all reference CSS vars):
- Semantic: `border`, `input`, `ring`, `background`, `foreground`, and `.DEFAULT/.foreground` pairs for `primary`, `secondary`, `destructive`, `muted`, `accent`, `popover`, `card` — each `hsl(var(--…))`.
- `status.{ok,issue,maintenance,sterilized,stale,unknown}` → `hsl(var(--status-…))`.
- `ivory.{bg,surface,border,borderMd,text,text2,text3,navy,green,greenMid,greenBg}` → `rgb(var(--ivory-…) / <alpha-value>)`, plus literal `ivory.{ok #16a34a, warn #d97706, err #dc2626, info #2563eb}`.
- `emergency.{bg,surface,border,borderMd,text,text2,accent,accentSoft,amber}` → `rgb(var(--emergency-…) / <alpha-value>)`.
- `offline.{bg,border,text}` → `rgb(var(--offline-…) / <alpha-value>)`.

**borderRadius:** `sm var(--radius-sm)`, `md var(--radius-md)`, `lg var(--radius)`, `xl var(--radius-xl)`, `2xl var(--radius-2xl)`, `pill var(--radius-pill)`.

**boxShadow:** `hero, card, card-hover, panel, surface, floating, modal, overlay` → the matching `var(--shadow-*)`.

**fontFamily:** `sans` = `-apple-system, system-ui, Heebo, Plus Jakarta Sans, Noto Sans Hebrew, Rubik, sans-serif`; `mono` = `IBM Plex Mono, ui-monospace, monospace`; `num` = `DM Mono, IBM Plex Mono, ui-monospace, monospace`.

**transitionTimingFunction:** `enter cubic-bezier(0.2,0.8,0.2,1)`, `reward cubic-bezier(0.34,1.56,0.64,1)`.
**transitionDuration:** `instant var(--motion-instant)`, `quick var(--motion-quick)`, `enter var(--motion-enter)`.

**keyframes:** `accordion-down`, `accordion-up`, `fade-in` (opacity + `translateY(4px)`), `page-enter` (same), `pro-rise` (`translateY(10px)`), `slide-in` (`translateX(-100%)→0`), `shimmer` (`backgroundPosition 200% 0 → -200% 0`).

## 5. Platform utilities

### 5.1 `src/lib/scan-affordance.ts`

The single scan-affordance gate for the whole app.

```ts
type ScanAffordance = "tab" | "fab" | "none";
function scanAffordance({ isNative, isTablet }): ScanAffordance {
  if (!isNative) return "none";     // web (any viewport) → no scan UI
  return isTablet ? "fab" : "tab";  // native tablet → FAB; native phone → flat scan tab
}
```

- **`tab`** — native phone: a flat, emphasized scan tab in the bottom bar.
- **`fab`** — native tablet: a floating scan button (iPad has no bottom bar).
- **`none`** — web, any viewport.

`useScanAffordance()` wraps the pure gate around live signals: `isTablet = useIsTabletViewport()`, `isNative = capacitorPlatform() !== "web"`. Every scan surface must read this — do not scatter platform checks.

### 5.2 `src/lib/use-tablet-viewport.ts`

Single source of truth for "is this a tablet-class viewport", forking the app shell (sidebar vs bottom tab bar) and the scan affordance so they can't disagree.

- Constants: `TABLET_MIN_WIDTH = 768`, `TABLET_MIN_HEIGHT = 500`, `TABLET_MEDIA_QUERY = "(min-width: 768px) and (min-height: 500px)"`.
- `isTabletViewport(w, h)` — pure predicate; tablet-class when **both** `w ≥ 768` **and** `h ≥ 500`. The min-height guard keeps large phones in landscape on the phone shell (a phone's short side never reaches 500px; the smallest tablet's is ~744pt), so it separates the classes in either orientation without reclassifying any iPad.
- `matchesTabletViewport()` — SSR-safe snapshot.
- `useIsTabletViewport()` — reactive hook, re-renders on the media-query boundary (rotate/resize).

> Companion: `src/hooks/use-is-desktop.ts` — `useIsDesktop()` returns true at `(min-width: 1024px)` (lg+), updating on resize. This is the desktop-breakpoint hook (the plan's `use-tablet-viewport` reference and this are distinct; both exist).

### 5.3 `src/hooks/use-focus-trap.ts`

Keyboard support for modal overlays. Shared by the equipment search overlay and the settings MoreSheet.

```ts
useFocusTrap({ active, containerRef, onEscape })
```

- **Contract:** while `active` is true, `Escape` calls `onEscape` and `Tab` is trapped within `containerRef` (cycling first↔last focusable). Focusable query: `a[href], button:not([disabled]), input, [tabindex]:not([tabindex="-1"])`.
- Initial focus is intentionally left to the caller (each overlay focuses a different first element).
- `onEscape` is read through a ref, so an inline callback doesn't re-subscribe the listener each render. Listener is removed when `active` flips false or on unmount.
