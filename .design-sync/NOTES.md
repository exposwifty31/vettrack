# design-sync notes â€” VetTrack

VetTrack is a full application, not a published component library, so this sync
runs in a non-standard configuration. Read this before re-syncing.

## Build configuration (how this sync works)

- **No `dist/` library entry.** Components are bundled from a generated entry,
  `src/design-system-entry.ts` (committed) â€” a hand-curated `export *` barrel of
  the visual components. Edit that file to add/remove components from the sync.
- **No shipped `.d.ts` tree.** Prop types are extracted directly from `.tsx`
  source. This requires three env vars on every `package-build.mjs` run:
  ```
  DS_SRC_GLOB="/Users/dan/vettrack-ship/src/**/*.ts,/Users/dan/vettrack-ship/src/**/*.tsx"
  DS_TS_BASEURL="/Users/dan/vettrack-ship"
  DS_TS_PATHS='{"@/*":["./src/*"],"@assets/*":["./docs/archive/2026/attached_assets/*"],"*":["./*"]}'
  ```
  These are consumed by patches in `.ds-sync/lib/dts.mjs` (see "Staged-script
  patches" below). Without them, every `.d.ts` collapses to `[key: string]: unknown`.
- **CSS is pre-compiled.** `cssEntry` points at `.design-sync/compiled.css`, a
  copy of the app's Tailwind build output (`dist/public/assets/index-*.css`).
  **Re-sync risk:** this copy goes stale. Before re-syncing, rebuild the app
  (`pnpm build`) and re-copy the latest `dist/public/assets/index-*.css` to
  `.design-sync/compiled.css`, or tokens/utilities added since the last app build
  will be missing.
- **`componentSrcMap`** in config.json pins all 110 componentâ†’source paths.
  Because `--entry` is set (bypassing synth-entry discovery), components are NOT
  auto-discovered â€” every component must have a `componentSrcMap` entry AND an
  export in `src/design-system-entry.ts`.

## Staged-script patches (NOT in git â€” re-apply on a fresh `.ds-sync/`)

`.ds-sync/` is gitignored and regenerated by copying the skill scripts. Two
patches were applied to the staged copies and must be re-applied after any fresh
stage:

1. **`.ds-sync/lib/bundle.mjs`** â€” in `tsconfigPathsPlugin`, the `exts` array had
   `''` (bare path) first, so a directory (`src/lib/api/`) matched before the
   file (`src/lib/api.ts`) when both exist. Fix: move `''` to the END of the
   `exts` list. Without this, the bundle fails with "Cannot read file ...: is a
   directory" for `@/lib/api`, `@/types`, `@/components/layout`.
2. **`.ds-sync/lib/dts.mjs`** â€” added `DS_SRC_GLOB` / `DS_TS_BASEURL` /
   `DS_TS_PATHS` env-var handling so prop interfaces extract from `.tsx` source
   (see env vars above). Without this, all props are `[key: string]: unknown`.

(These could be moved into `.design-sync/overrides/*.mjs`, but the overrides
mechanism needs esbuild/ts-morph resolvable from the repo root, which this repo
doesn't have â€” so direct staged-script edits are the working approach.)

## Render verification

- **Previews were NOT render-verified** (validated with `--no-render-check` per
  user request). All 110 components ship as honest "floor cards" (typographic
  name blocks) â€” they are fully functional and have real `.d.ts` + `.prompt.md`,

  re-run the build, then validate (ideally with the render check working).

## Component grouping

- UI primitives landed in the **`general`** group because `src/components/ui/` is
  a generic dir name the converter filters from grouping. Acceptable but flat; to
  regroup, point `cfg.docsMap.<Name>` at a stub `.md` with `category:` frontmatter.

## Fonts

- Brand fonts (Plus Jakarta Sans, Heebo, Noto Sans Hebrew, Rubik, DM Mono, IBM
  Plex Mono) are runtime/host-served (declared via `runtimeFontPrefixes`), NOT
  bundled. Designs render in fallback fonts unless the host page loads them.

## Theme / palette (default = clinical/indigo)

The DS ships a **theme matrix** keyed on `data-color-theme` (each with a `.dark`
variant). Bare `:root` and `:root[data-color-theme=clinical]` are identical â€” the
**clinical** default: cool iOS-gray surface (`--ivory-bg: 242 242 247` = `#f2f2f7`)
+ **indigo** primary (`--primary: 243 75% 59%` â‰ˆ `#5048e5`). Note `--ivory-green`
is indigo (`#4f46e5`) under clinical, green only under forest.
`:root[data-color-theme=forest]` is the deep forest-green primary (`142 72% 29%`).
The design tool renders previews under bare `:root`, i.e. **clinical/indigo**.

- **conventions.md drift caught & fixed 2026-07-03:** the header still described the
  old "warm off-white / forest-green" look. Corrected to clinical/indigo + a note
  that forest is opt-in via `data-color-theme=forest`. The conventions-header
  validation step re-checks these color claims against the fresh `_ds_bundle.css`
  `:root` values each re-sync â€” if the app flips the default theme again, expect drift
  to surface there.

## Re-sync checklist

1. `pnpm build` â†’ copy `dist/public/assets/index-*.css` â†’ `.design-sync/compiled.css`
2. Re-stage `.ds-sync/` and re-apply the two staged-script patches above
3. Run `resync.mjs --remote .design-sync/.cache/remote-sync.json` with the three `DS_*` env vars set
4. Validate (turn on render check if possible)
5. Upload per atomic path (re-sync)

**Important:** Save the full `_ds_sync.json` content from `DesignSync(get_file)` to
`.design-sync/.cache/remote-sync.json` before running the driver. The driver needs
all fields (`sourceKeys`, `sourceHashes`, `bundleSha12`, etc.) to produce a useful
diff â€” saving only partial content results in "anchor malformed" and forces a full
re-upload of all 110 components instead of a targeted diff.

**Repo path:** The DS_SRC_GLOB / DS_TS_BASEURL env vars use `/Users/dan/vettrack-ship/`
(not `/Users/dan/vettrack/`).

## Re-sync risks

- **compiled.css goes stale** â€” any commit touching `src/index.css` or Tailwind
  config requires `pnpm build` + CSS re-copy before re-syncing.
- **Previews are floor cards only** â€” no authored `.design-sync/previews/` files;
  all 110 components render as typographic name blocks.
- **Staged-script patches are lost on fresh `.ds-sync/`** â€” must re-apply both
  patches every time `.ds-sync/` is regenerated.
- **Default theme can silently flip** â€” the design tool renders the `:root`
  (`clinical`) palette. A theme change in `src/index.css` / tailwind config shifts
  `--primary` / `--ivory-*`; the conventions-header validation catches stale color
  copy, but re-verify it after any such change (see "Theme / palette").
