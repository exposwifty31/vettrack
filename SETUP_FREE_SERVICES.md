# Setup Guide: Free Open-Source Services for VetTrack\n\n## What's Included\n\nAll services are 100% free, self-hosted alternatives:\n\n```\n✅ Meilisearch      → Patient search (<100ms)\n✅ Kafka            → Event sourcing + audit trail\n✅ Socket.io        → Real-time collaboration + code blue alerts\n✅ Grafana          → Clinic metrics dashboard\n✅ Elasticsearch    → Centralized logging\n✅ Logstash         → Log aggregation\n✅ Kibana           → Log visualization\n✅ Mattermost       → Team chat\n✅ Postgres + TimescaleDB → Time-series analytics\n```\n\n---\n\n## Option 1: Local Development (Docker Compose)\n\n### Start All Services\n\n```bash\n# From repo root\ndocker-compose -f docker-compose.free-services.yml up -d\n```\n\n### Verify Services Are Running\n\n```bash\n# Check all containers\ndocker ps\n\n# Check logs\ndocker logs vettrack-meilisearch\ndocker logs vettrack-kafka\ndocker logs vettrack-grafana\n```\n\n### Access Web Interfaces\n\n```\nMeilisearch:  http://localhost:7700\nGrafana:      http://localhost:3001 (admin/admin)\nKibana:       http://localhost:5601\nMattermost:   http://localhost:8065\nElasticsearch: http://localhost:9200\n```\n\n### Environment Variables\n\nCreate `.env.local`:\n\n```bash\n# Meilisearch\nMEILISEARCH_HOST=http://meilisearch:7700\nMEILISEARCH_API_KEY=masterKey\n\n# Kafka\nKAFKA_BROKERS=kafka:9092\n\n# Postgres (for Grafana + Logstash)\nPOSTGRES_USER=postgres\nPOSTGRES_PASSWORD=postgres\nDATABASE_URL=postgresql://postgres:postgres@postgres:5432/vettrack\n\n# Grafana\nGRAFANA_PASSWORD=admin\n```\n\n---\n\n## Option 2: Deploy to Railway\n\n### Step 1: Create Services\n\n```bash\n# Deploy Meilisearch\nrailway service create --name meilisearch --image getmeili/meilisearch:latest\n\n# Deploy Kafka (with Zookeeper)\nrailway service create --name zookeeper --image confluentinc/cp-zookeeper:7.5.0\nrailway service create --name kafka --image confluentinc/cp-kafka:7.5.0\n\n# Deploy Grafana\nrailway service create --name grafana --image grafana/grafana:latest\n\n# Deploy ELK Stack\nrailway service create --name elasticsearch --image docker.elastic.co/elasticsearch/elasticsearch:8.0.0\nrailway service create --name logstash --image docker.elastic.co/logstash/logstash:8.0.0\nrailway service create --name kibana --image docker.elastic.co/kibana/kibana:8.0.0\n```\n\n### Step 2: Configure Environment Variables\n\nFor each service in Railway:\n\n**Meilisearch**:\n- `MEILI_ENV`: production\n- `MEILI_MASTER_KEY`: (set strong key)\n\n**Kafka**:\n- `KAFKA_ZOOKEEPER_CONNECT`: zookeeper:2181\n- `KAFKA_BROKER_ID`: 1\n- `KAFKA_ADVERTISED_LISTENERS`: PLAINTEXT://kafka:29092,PLAINTEXT_HOST://kafka:9092\n\n**Grafana**:\n- `GF_SECURITY_ADMIN_PASSWORD`: (set password)\n- `GF_SECURITY_ADMIN_USER`: admin\n\n**Elasticsearch**:\n- `discovery.type`: single-node\n- `xpack.security.enabled`: false\n\n### Step 3: Create Private Network\n\nIn Railway → Project Settings → Networking:\n- Enable private networking\n- Services can reach each other via `service-name:port`\n\n---\n\n## Integration with VetTrack\n\n### 1. Update package.json\n\nAdd dependencies:\n\n```bash\npnpm install meilisearch kafkajs socket.io\n```\n\n### 2. Wire Up in server/index.ts\n\n```typescript\nimport { initKafkaProducer } from './lib/kafka-client.js';\nimport { initRealtimeServer } from './lib/realtime-server.js';\nimport { initializeSearchIndices } from './lib/meilisearch-client.js';\n\n// In startup\nawait initKafkaProducer();\nawait initializeSearchIndices();\n\nconst httpServer = createServer(app);\nconst io = initRealtimeServer(httpServer);\n\nhttpServer.listen(3001);\n```\n\n### 3. Add Search Route\n\nIn server/app/routes.ts:\n\n```typescript\nimport searchRoutes from '../routes/search.js';\n\napp.use('/api/search', searchRoutes);\n```\n\n### 4. Index Patients on Create/Update\n\nIn your patient mutation handler:\n\n```typescript\nimport { indexPatient } from '../lib/meilisearch-client.js';\nimport { logAuditEvent } from '../lib/kafka-client.js';\n\n// After creating patient in database\nawait indexPatient(patient);\nawait logAuditEvent('patient-created', 'patient', patient.id, clinicId, userId);\n```\n\n---\n\n## Usage Examples\n\n### Search Patients\n\n```bash\ncurl \"http://localhost:3001/api/search/patients?q=john&limit=10\"\n\n# Response:\n# {\n#   \"query\": \"john\",\n#   \"results\": [\n#     { \"id\": \"123\", \"name\": \"John Smith\", \"email\": \"john@clinic.com\", \"phone\": \"555-1234\" },\n#     { \"id\": \"456\", \"name\": \"John Doe\", \"email\": \"johndoe@clinic.com\", \"phone\": \"555-5678\" }\n#   ]\n# }\n```\n\n### Publish Event to Kafka\n\n```typescript\nimport { publishEvent } from './lib/kafka-client.js';\n\nawait publishEvent('patient-admitted', {\n  patientId: '123',\n  clinicId: 'clinic-456',\n  timestamp: new Date(),\n});\n```\n\n### Broadcast Code Blue Alert\n\n```typescript\nimport { broadcastCodeBlue } from './lib/realtime-server.js';\n\nawait broadcastCodeBlue('clinic-456', 'Surgery Room 2', 'dr-smith');\n// All staff in clinic receive real-time notification\n```\n\n### Log Audit Event\n\n```typescript\nimport { logAuditEvent } from './lib/kafka-client.js';\n\nawait logAuditEvent(\n  'equipment-checked-out',\n  'equipment',\n  'eq-789',\n  'clinic-456',\n  'user-123',\n  { location: 'Exam Room 1', timestamp: new Date() }\n);\n```\n\n---\n\n## Monitoring\n\n### Grafana Dashboards\n\n1. Add Postgres data source (points to your database)\n2. Create dashboard with TimescaleDB queries:\n\n```sql\n-- Real-time occupancy\nSELECT\n  time_bucket('5 minutes', created_at) as time,\n  COUNT(*) as occupied_rooms\nFROM room_occupancy\nWHERE clinic_id = 'clinic-456' AND status = 'occupied'\nGROUP BY time\nORDER BY time DESC;\n\n-- Revenue by hour\nSELECT\n  time_bucket('1 hour', created_at) as hour,\n  SUM(amount) as revenue\nFROM billing\nWHERE clinic_id = 'clinic-456'\nGROUP BY hour\nORDER BY hour DESC;\n```\n\n### Kibana Logs\n\n1. Create index pattern: `vettrack-*`\n2. View logs from all services\n3. Set up alerts on error rates\n\n---\n\n## Cleanup\n\n### Stop Local Services\n\n```bash\ndocker-compose -f docker-compose.free-services.yml down\n\n# Remove volumes (careful!)\ndocker-compose -f docker-compose.free-services.yml down -v\n```\n\n### Remove from Railway\n\n```bash\nrailway service remove meilisearch\nrailway service remove kafka\nrailway service remove zookeeper\n# etc.\n```\n\n---\n\n## Troubleshooting\n\n### Meilisearch not reachable\n\n```bash\n# Check if running\ncurl http://localhost:7700/health\n\n# Check logs\ndocker logs vettrack-meilisearch\n```\n\n### Kafka connection refused\n\n```bash\n# Verify Zookeeper is running first\ndocker logs vettrack-zookeeper\n\n# Then check Kafka\ndocker logs vettrack-kafka\n\n# List topics\ndocker exec vettrack-kafka kafka-topics --list --bootstrap-server localhost:9092\n```\n\n### Grafana can't connect to Postgres\n\n1. Verify Postgres is running: `docker ps | grep postgres`\n2. Check connection string uses private network name: `postgres:5432` (not localhost)\n3. Verify credentials in environment variables\n\n---\n\n## Next Steps\n\n1. **Start local** (docker-compose)\n2. **Test search, Kafka, real-time** in development\n3. **Deploy to Railway** when ready\n4. **Monitor dashboards** in Grafana\n5. **Scale as needed** (add more instances of each service)\n\n---\n\n## Cost Comparison\n\n| Service | Managed Cost | Free Self-Hosted |\n|---------|---|---|\n| Search (Elasticsearch) | $150/mo | Free |\n| Message Queue (Kafka) | $100/mo | Free |\n| Analytics (Timescale) | $150/mo | Free (PostgreSQL extension) |\n| Dashboards (Grafana Cloud) | $100/mo | Free |\n| Logging (ELK Cloud) | $200/mo | Free |\n| **Total** | **$700/mo** | **Free** |\n\nOnce deployed to Railway: +~$115/month in compute (vs +$700/month for managed services)\n\n**Savings: $585/month** ✅\n"
